Communication API Documentation

POST /api/communication/comment

Add a comment to a SPA item in Bitrix24. Requires authentication.

Request Body

{
    "entity_id": 1038,
    "comment": "This is a test comment"
}

Response (200 OK)

{
    "success": true,
    "result": { "ID": 123 }
}

Response (400 Bad Request)

{
    "success": false,
    "error": "OWNER_NOT_FOUND",
    "error_description": "Owner was not found"
}

POST /api/communication/lead

Store a new lead in Bitrix24. No authentication required.

Request Body

{ 
    "name": "John Doe", 
    "email": "john@example.com", 
    "phone": "+1234567890", 
    "message": "Interested in your services",
    "available_time": "Tomorrow 10:00 - 12:00"
}

Response (200 OK)

{ 
    "success": true, 
    "result": 101 
}

Response (400 Bad Request)

{ 
    "success": false, 
    "error": "Invalid email", 
    "error_description": null 
}

Notes

  • available_time is optional and represents the preferred time for contact.

POST /api/communication/contact

Add a new contact to Bitrix24 for unauthenticated users.

Request Body

{
    "first_name": "John",
    "second_name": "Michael",
    "last_name": "Doe",
    "email": "john@example.com",
    "phone": "+1234567890"
}

Response (200 OK)

{
    "success": true,
    "contact_id": 201
}

Response (400 Bad Request)

{
    "success": false,
    "error": "EMAIL_ALREADY_EXISTS",
    "error_description": null
}
