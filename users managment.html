User Management API Documentation

GET /api/users

Description: List users with search and pagination.

Query Parameters

ParameterTypeDescription
pageintegerPage number (default 1)
per_pageintegerItems per page (default 10)
searchstringSearch by name or email

Response

{
    "data": [
        {
            "id": 1,
            "name": "John Doe",
            "email": "john@example.com",
            "role": "agent",
            "agent_id": null,
            "team_lead_id": 2
        }
    ],
    "total": 1,
    "page": 1,
    "per_page": 10
}

POST /api/users

Description: Create a new user (permissions depend on role).

Request Body

{
    "name": "Jane Doe",
    "email": "jane@example.com",
    "password": "password123",
    "role": "agent"
}

Response (201 Created)

{
    "id": 5,
    "name": "Jane Doe",
    "email": "jane@example.com",
    "role": "agent",
    "agent_id": null,
    "team_lead_id": 2
}

GET /api/users/{id}

Description: Get a specific user by ID.

Response

{
    "id": 5,
    "name": "Jane Doe",
    "email": "jane@example.com",
    "role": "agent",
    "agent_id": null,
    "team_lead_id": 2
}

PUT /api/users/{id}

Description: Update user information.

Request Body (optional fields)

{
    "name": "Jane Smith",
    "email": "janesmith@example.com",
    "password": "newpassword123",
    "role": "agent"
}

Response

{
    "id": 5,
    "name": "Jane Smith",
    "email": "janesmith@example.com",
    "role": "agent",
    "agent_id": null,
    "team_lead_id": 2
}

DELETE /api/users/{id}

Description: Delete a user.

Response

{
    "message": "Deleted successfully"
}
