Auth API Documentation

POST /api/register

Description: Register a new client user. Only role "client" is allowed.

Request Body

{
    "name": "John Doe",
    "email": "john@example.com",
    "password": "password123",
    "phone": "01123456789"
}

Response (201 Created)

{
    "message": "Account created. Please verify your email.",
    "user": {
        "id": 10,
        "name": "John Doe",
        "email": "john@example.com",
        "role": "client",
        "agent_id": null,
        "team_lead_id": null,
        "phone": "01123456789",
        "email_verified_at": null,
        "created_at": "2026-01-07T12:00:00",
        "updated_at": "2026-01-07T12:00:00"
    }
}

POST /api/login

Description: Login with email and password. Email must be verified.

Request Body

{
    "email": "john@example.com",
    "password": "password123"
}

Response (200 OK)

{
    "token": "2|abcd1234efgh5678ijkl9012mnop3456qrst",
    "user": {
        "id": 10,
        "name": "John Doe",
        "email": "john@example.com",
        "role": "client",
        "agent_id": null,
        "team_lead_id": null,
        "phone": "01123456789",
        "email_verified_at": "2026-01-07T12:30:00",
        "created_at": "2026-01-07T12:00:00",
        "updated_at": "2026-01-07T12:30:00"
    }
}

Response (401 Unauthorized)

{
    "message": "Invalid credentials"
}

Response (403 Forbidden - email not verified)

{
    "message": "Please verify your email before login"
}

POST /api/logout

Description: Revoke the current API token. Requires authentication.

Headers

HeaderValue
AuthorizationBearer <token>

Response (200 OK)

{
    "message": "Logged out"
}

GET /api/email/verify/{id}/{hash}

Description: Verify a newly registered user's email.

Response (200 OK)

{
    "message": "Email verified successfully"
}

Response (403 Invalid link)

{
    "message": "Invalid verification link"
}

Response (404 User not found)

{
    "message": "User not found"
}

POST /api/email/verification-notification

Description: Resend the email verification link. Requires authentication.

Headers

HeaderValue
AuthorizationBearer <token>

Response (200 OK)

{
    "message": "Verification link sent"
}

POST /api/forgot-password

Description: Send a password reset OTP to the user's email.

Request Body

{
    "email": "john@example.com"
}

Response (200 OK)

{
    "message": "OTP sent to email"
}

POST /api/verify-otp

Description: Verify the OTP sent for password reset.

Request Body

{
    "email": "john@example.com",
    "otp": "123456"
}

Response (200 OK)

{
    "message": "OTP verified"
}

Response (400 Invalid OTP)

{
    "message": "Invalid OTP"
}

Response (400 Expired OTP)

{
    "message": "OTP expired"
}

POST /api/reset-password

Description: Reset password using OTP.

Request Body

{
    "email": "john@example.com",
    "otp": "123456",
    "password": "newpassword123"
}

Response (200 OK)

{
    "message": "Password reset successful"
}

Response (400 Invalid/Expired OTP)

{
    "message": "Invalid or expired OTP"
}
