Auth API Documentation

POST /api/register

Register new client user (role is forced to client). Supports profile image upload.

Request (multipart/form-data)

{
  "name": "John Doe",
  "email": "john@example.com",
  "password": "password123",
  "phone": "01123456789",
  "image": "file (optional)"
}

Response (201 Created)

{
  "message": "Account created. Please verify your email.",
  "user": {
    "id": 10,
    "name": "John Doe",
    "email": "john@example.com",
    "role": "client",
    "phone": "01123456789",
    "image": "users/avatar.png",
    "email_verified_at": null
  }
}

POST /api/login

Login using email & password. Email must be verified.

Request

{
  "email": "john@example.com",
  "password": "password123"
}

Response (200 OK)

{
  "token": "1|sanctumtoken...",
  "user": {
    "id": 10,
    "name": "John Doe",
    "email": "john@example.com",
    "role": "client"
  }
}

Errors

401 - Invalid credentials
403 - Please verify your email before login

POST /api/logout

Revoke current Sanctum token

Headers

Authorization Bearer <token>

Response

{ "message": "Logged out" }

POST /api/forgot-password

Send OTP to email for password reset

Request

{ "email": "john@example.com" }

Response

{ "message": "OTP sent to email" }

POST /api/verify-otp

Request

{
  "email": "john@example.com",
  "otp": "123456"
}

Responses

200 - OTP verified
400 - Invalid OTP
400 - OTP expired

POST /api/reset-password

Request

{
  "email": "john@example.com",
  "otp": "123456",
  "password": "newpassword123"
}

Response

{ "message": "Password reset successful" }

GET /api/profile

Get authenticated user's profile data.

Headers

Authorization Bearer <token>

Response (200 OK)

{
  "user": {
    "id": 10,
    "name": "John Doe",
    "email": "john@example.com",
    "role": "agent",
    "phone": "01123456789",
    "instagram_link": "...",
    "tiktok_link": "...",
    "whatsapp_link": "...",
    "bio": "Expert in luxury villas.",
    "image": "users/avatar.png",
    "average_rating": 4.5
  }
}

POST /api/update-profile

Update authenticated user's profile. Role cannot be updated.

Headers

Authorization Bearer <token>

Request (multipart/form-data)

{
  "name": "John Doe",
  "phone": "01123456789",
  "password": "optional",
  "instagram_link": "https://instagram.com/...",
  "tiktok_link": "https://tiktok.com/...",
  "whatsapp_link": "https://wa.me/...",
  "bio": "New bio content",
  "image": "file (optional)"
}

Response

{
  "message": "Profile updated successfully",
  "user": { ... }
}
