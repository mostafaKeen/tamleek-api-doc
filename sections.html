GET/api/sections/{idOrName}

üåç Public Access - No authentication required

Get a specific section by its numeric ID or unique section_name.

üí° Frontend Tip: Use the name (e.g., /api/sections/services) to avoid hardcoding database IDs.

Response (200 OK)

{
    "id": 1,
    "section_name": "about-us",
    "p1": "...",
    "p2": "...",
    "p3": "...",
    "subsections": [...]
}

POST/api/sections

üîí Admin Only - Sanctum token + Admin role

Create a new section.

Request Body

{
    "section_name": "services",
    "p1": "Leading real estate services...",
    "p2": "Optional second paragraph",
    "p3": "Optional third paragraph"
}

Validation Rules

Field Rules Description
section_name required, string, unique Unique slug/name for the section
p1 nullable, string First block of text

GET/api/subsections

üåç Public Access - No authentication required

List subsections with optional filtering.

Query Parameters

Parameter Type Description
section_id integer Filter by Section ID
section_name string Filter by Section Name (e.g., ?section_name=about-us)

POST/api/subsections

üîí Admin Only - Sanctum token + Admin role

Create a new subsection linked to a section.

Request Body

{
    "section_id": 1,
    "content": "This is the HTML or long text content of the subsection."
}

Validation Rules

Field Rules Description
section_id required, exists:sections,id ID of the parent section
content required, string Long text or HTML content

DELETE/api/sections/{id}

üîí Admin Only

Deleting a section will cascade delete all its subsections.
