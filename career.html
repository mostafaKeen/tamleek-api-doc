Career API Documentation

POST /api/career/store

Description: Submit a career application (SPA 1042) including CV and photo. Multiple file upload supported.

Request Body (multipart/form-data)

{
    "full_name": "Jane Doe",
    "phone": "+11234567890",
    "email": "jane@example.com",
    "experience": 3,                  // optional, in years
    "cv": (PDF/DOC/DOCX file),        // required
    "photo": (JPG/PNG file),          // required
    "linkedin": "https://linkedin.com/in/jane",   // optional
    "facebook": "https://facebook.com/jane",     // optional
    "instagram": "https://instagram.com/jane",   // optional
    "twitter": "https://twitter.com/jane",       // optional
    "tiktok": "https://tiktok.com/@jane",        // optional
    "reason": "Why I am a good fit for this role" // required
}

Response (Success - 200 OK)

{
    "success": true,
    "item_id": 12345
}

Response (Validation Error - 400 Bad Request)

{
    "success": false,
    "error": "validation_error",
    "error_description": {
        "full_name": ["The full name field is required."],
        "cv": ["The cv must be a file of type: pdf, doc, docx."],
        ...
    },
    "bitrix_response": {...}  // optional response from Bitrix API
}

Response (Bitrix API Error - 400 Bad Request)

{
    "success": false,
    "error": "ERROR_CODE",
    "error_description": "Detailed error from Bitrix",
    "bitrix_response": {...}
}

Notes

  • File fields (`cv` and `photo`) are encoded in base64 and sent to Bitrix SPA field `ufcrm_10_1769065094`.
  • `experience` is optional but must be an integer â‰¥ 0 if provided.
  • URLs for social media fields must be valid URLs if provided.
  • All required fields must be present; otherwise, validation error is returned.
